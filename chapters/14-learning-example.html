Learning Example: Skill Acquisition Using HEX

A phased walkthrough that shows how to diagnose learning problems and make the right structural move.

HOW TO USE THIS EXAMPLE

This walkthrough shows how to use HEX on a real learning goal. Treat it like a diagnostic loop: define the system, map the axes, identify failure mode, locate authority, and make one small change.

Phase 1 ‚Äî Define the system

System
The learner (mind + body + habits)
Goal
Increase capability by reorganizing capacity under constraint (not ‚Äúadding facts‚Äù).

Rule: Learning is structural change. If there is no structural change, there is no learning.

Phase 3 ‚Äî Identify state and failure mode

State check: Is the system stable, strained, or in flow?

FAILURE MODES TO WATCH
  • Overload: too much difficulty/pace/pressure for current coherence
  • Rigidity: practice becomes tight, perfectionistic, or repetitive without adaptation
  • Decay: forgetting, inconsistency, or loss of motivation over time

If progress feels random, coherence is the first suspect. If progress is costly, load is too high. If gains vanish, persistence is unprotected.

Phase 4 ‚Äî Locate the hinge

HINGE SIGNALS
  • Repeated failure at the same point
  • Plateau despite effort
  • Escalating cost for the same progress
  • Workarounds multiplying

Interpretation: the system is asking for reconfiguration, not more effort of the same kind.

Phase 5 ‚Äî Authority relocation (the move)

AUTHORITY CHECK
  • What variable actually controls progress right now?
  • Who/what is setting the pace: the task, ego, environment, or feedback quality?
  • Are decisions and consequences aligned?
TRANSFORM MOVE
  • Lower load before demanding insight (reduce difficulty, slow pace, reduce pressure)
  • Increase coherence before increasing reps (clear model, better feedback, better sequencing)
  • Protect persistence (sleep, spaced repetition, consistent schedule)

Rule: Authority must relocate‚Äînever dissolve. If you lose the ability to choose or rebind, stop and return to role basics.

  • ‚ÄúI understand what‚Äôs happening.‚Äù
  • ‚ÄúI can predict errors and correct them.‚Äù
  • ‚ÄúThis skill belongs to me now.‚Äù
